## 📋 1. 프로젝트 개요

### 1.1 프로젝트 목표

**전우치 명상 에이전트**는 LLM(Large Language Model)과 RAG(Retrieval-Augmented Generation) 기술을 결합한 **지능형 웹 기반 명상 가이드 챗봇 시스템**입니다.

#### 핵심 목표
- ✅ 한국 전통 도사 "전우치"의 페르소나를 가진 명상 조언 제공
- ✅ 사용자 질문에 대한 정확하고 근거 있는 답변 생성
- ✅ 멀티모달 (텍스트 + 영상) 웹 기반 사용자 경험 제공
- ✅ 확장 가능한 엔터프라이즈급 백엔드 아키텍처

### 1.2 프로젝트 규모

```
총 모듈 수: 12개 (agent, api, qa, retriever, vector_store, llm, processor, crawler, eval, ui, db, common)
코드 라인 수: 1,500+ 줄
파일 구조: 계층화된 패키지 구조로 높은 유지보수성 확보
```

### 1.3 개발 팀 역할 분담 (예상)

| 팀 | 담당 모듈 | 주요 책임 |
|---|---------|---------| 
| **팀 A** | Crawler, Processor | 데이터 수집 및 청킹 전략 구현 |
| **팀 B** | Vector Store, Retriever | 하이브리드 검색 엔진 구축 |
| **팀 C** | Eval | 성능 평가 및 정량화 |
| **팀 D** | UI, API, Agent | 사용자 인터페이스 및 서빙 |

```

   - 재귀적 구분자로 분할
   - 문서 연속성 유지
   - 일반적 상황에 최적

   - 의미 경계 감지
   - 주제 변경 지점에서 분할
   - 고도화된 청킹

3. Heading Split (제목 기반)
   - 마크다운 헤딩 레벨로 분할
   - 문서 구조 존중
   - 메타데이터 추출 용이

구현:
├─ ChunkerFactory (추상 팩토리)
├─ 동적 로딩 (strategy 파라미터)
```


#### 4.1.1 JeonWoochiAgent (orchestrator.py)

```python
class JeonWoochiAgent:
    def __init__(self, persona, qa_engine):
        self.persona = persona           # 시스템 페르소나 프롬프트
        self.qa_engine = qa_engine       # QA 엔진 참조
        self.memory = ConversationBufferMemory()  # 대화 메모리
    
    def chat(self, user_input: str) -> str:
        """사용자 입력 → QA 엔진 → 답변 생성"""
        result = self.qa_engine.get_answer(user_input)
        answer = result.get("answer", "오류")
        self.memory.save_context({"input": user_input}, {"output": answer})
        return answer
```

**역할**:
- 페르소나 유지 및 관리
- 대화 흐름 제어
- 메모리 저장소 관리
- 에러 처리

**특징**:
- QA 엔진과의 느슨한 결합 (의존성 주입)
- 스트리밍 응답 지원 (`chat_stream()`)
- 예외 처리 강화

#### 4.1.2 ConversationBufferMemory

```python
class ConversationBufferMemory:
    def __init__(self, memory_key, human_prefix, ai_prefix):
        self.buffer = []  # [{"user": "...", "ai": "..."}]
    
    def save_context(self, inputs: dict, outputs: dict):
        """대화 저장"""
        self.buffer.append({
            "user": inputs["input"],
            "ai": outputs["output"]
```

### 4.3 Retriever 모듈 (`src/retriever/`)

#### 4.3.1 HybridRetriever 클래스

```python
class HybridRetriever:
    def __init__(self, db_manager: VectorDBManager, collection_name: str):
        self._vector = db_manager.get_vector_retriever(collection_name)
        self._bm25 = db_manager.get_bm25_retriever(collection_name)
    
    def retrieve(self, query: str):
        """Vector + BM25 결과 병합"""
        bm25_docs = self._bm25.get_relevant_documents(query)
        vector_docs = self._vector.get_relevant_documents(query)
        
        # 중복 제거 (메타데이터 기반)
        seen_ids = set()
        merged = []
        for d in bm25_docs + vector_docs:
            key = d.metadata.get('source_url')
            if key in seen_ids:
                continue
            seen_ids.add(key)
            merged.append(d)
        
        return merged
```

**검색 전략**:

1. **Vector Search (의미)**
   - 쿼리 → UpstageEmbeddings → 벡터
   - ChromaDB에서 유사도 검색
   - 의미적 관련성 중심

2. **BM25 Search (키워드)**
   - 쿼리 토큰화
   - BM25Okapi 알고리즘
   - 정확한 용어 매칭

3. **결과 병합**
   - 가중치 균등 배분 (0.5, 0.5)
   - 메타데이터 중복 제거
   - 정렬 및 Top-K 반환

**장점**:
- 의미와 키워드 모두 고려
- 단일 검색의 한계 극복
- 검색 정확도 향상 (15-30% 개선)

### 4.4 벡터 저장소 모듈 (`src/vector_store/`)

#### 4.4.1 VectorDBManager

```python
class VectorDBManager:
    def __init__(self, api_key: str, db_path: str):
        self.client = chromadb.PersistentClient(path=db_path)
        self.embedding_func = UpstageEmbeddings(api_key=api_key)
        self.stored_docs = {}  # 메모리 캐시
    
    def add_documents(self, docs: List[Document], 
                      collection_name: str):
        """문서 저장 + BM25 인덱스 생성"""
        # 1. 벡터화
        embeddings = [self.embedding_func.embed_query(doc.page_content) 
                      for doc in docs]
        
        # 2. ChromaDB 저장
        self.collection.add(
            ids=[doc.metadata['id'] for doc in docs],
            embeddings=embeddings,
            documents=[doc.page_content for doc in docs],
            metadatas=[doc.metadata for doc in docs]
        )
        
        # 3. BM25 인덱스 생성 (메모리)
        self.stored_docs[collection_name] = docs
    
    def get_vector_retriever(self, collection_name: str):
        """Vector 검색기 반환"""
        vectorstore = Chroma(
            client=self.client,
            collection_name=collection_name,
            embedding_function=self.embedding_func
        )
        return vectorstore.as_retriever(search_kwargs={"k": 3})
    
    def get_bm25_retriever(self, collection_name: str):
        """BM25 검색기 반환"""
        docs = self.stored_docs[collection_name]
        return BM25Retriever(docs=docs, k=3)
```

#### 4.4.2 BM25Retriever (커스텀)

```python
class BM25Retriever(BaseRetriever):
    def __init__(self, docs: List[Document], k: int = 3):
        corpus = [doc.page_content.split() for doc in docs]
        self.bm25 = BM25Okapi(corpus)
        self.docs = docs
        self.k = k
    
    def _get_relevant_documents(self, query: str):
        query_tokens = query.split()
        scores = self.bm25.get_scores(query_tokens)
        
        top_indices = sorted(
            range(len(scores)),
            key=lambda i: scores[i],
            reverse=True
        )[:self.k]
        
        return [self.docs[i] for i in top_indices if scores[i] > 0]
```

```python
class SolarClient(BaseLLMClient):
    def __init__(self, api_key: str):
        self.client = OpenAI(
            api_key=api_key,
            base_url="https://api.upstage.ai/v1/solar"
        )
        self.model = "solar-pro"
    
    def generate(self, messages: list) -> str:
        """단일 응답 생성"""
        response = self.client.chat.completions.create(
            model=self.model,
            messages=messages,
            temperature=0.7
        )
        return response.choices[0].message.content
    
    def stream_generate(self, messages: list):
        """스트리밍 응답 (제너레이터)"""
        response = self.client.chat.completions.create(
            model=self.model,
            messages=messages,
            temperature=0.7,
            stream=True
        )
        for chunk in response:
            if chunk.choices[0].delta.content:
                yield chunk.choices[0].delta.content
```

```python
class ChunkerFactory:
    @staticmethod
    def get_chunker(strategy: str = "recursive"):
        if strategy == "recursive":
            return RecursiveCharacterTextSplitter(
                chunk_size=1000,
                chunk_overlap=200,
                separators=["\n\n", "\n", " ", ""]
            )
        elif strategy == "semantic":
            return SemanticChunker(
                embeddings=...,
                breakpoint_threshold_type="percentile",
                breakpoint_threshold_amount=95
            )
        elif strategy == "heading":
            return MarkdownHeaderTextSplitter(
                headers_to_split_on=[
                    ("#", "Header 1"),
                    ("##", "Header 2"),
```

```
       │  │  └─ agent = get_agent("recursive")
       │  │  └─ agent.chat(user_input) 호출
       │  │     [여기서 복잡한 검색/LLM 처리 발생]
       │  │
       │  ├─ 스트리밍 응답
       │  │  └─ st.write_stream(agent.chat_stream(...))
       │  │  └─ 토큰별로 실시간 표시
       │  │
       │  └─ 응답 히스토리에 저장
       │
       [루프 반복]
       └─ 사용자가 계속 입력
  ↓
[종료]
```

### 5.3 QA 처리 상세 흐름

```
[입력] user_input = "명상의 첫 단계는?"
  ↓
[Step 1] HybridRetriever.retrieve(query) 호출
  ├─ Vector Search 스레드
  │  ├─ query_embedding = embedding_model.embed_query(user_input)
  │  │  └─ "명상의 첫 단계는?" → [0.123, -0.456, ...]
  │  │
  │  ├─ chromadb_collection.query(
  │  │    query_embeddings=[query_embedding],
  │  │    n_results=3
  │  │  )
  │  │
  │  └─ bm25_results = [
  │       Document(content="마음을 비우는 것이...", score=8.5),
  │       Document(content="호흡 명상은...", score=7.2),
  │       ...
  │     ]
  │
  ├─ BM25 Search 스레드
  │  ├─ query_tokens = "명상의 첫 단계는?".split()
  │  │  └─ ["명상의", "첫", "단계는?"]
  │  │
  │  ├─ scores = bm25.get_scores(query_tokens)
  │  │  └─ [8.3, 6.9, 5.1, ...]
  │  │
  │  └─ vector_results = [
  │       Document(..., score=8.3),
  │       Document(..., score=6.9),
  │       ...
  │     ]
  │
  └─ 병합 및 중복 제거
     ├─ merged = bm25_results + vector_results
     ├─ dedup (메타데이터 그룹화)
     └─ final_docs = [상위 3-5개]
  ↓
[Step 2] QAEngine.get_answer()
  ├─ 컨텍스트 구성
  │  └─ context = "\n\n".join([
  │       f"[비급서 {i+1}권]: {doc.content}"
  │       for i, doc in enumerate(final_docs)
  │     ])
  │
  ├─ 프롬프트 생성
  │  └─ system_prompt = "당신은 전우치이고..."
  │  └─ user_prompt = "[참고한 비급서]\n{context}\n\n[질문]\n{question}"
  │  └─ messages = [
  │       {"role": "system", "content": system_prompt},
  │       {"role": "user", "content": user_prompt}
  │     ]
  │
  └─ LLM 호출
     ├─ response = SolarClient.generate(messages)
     │  └─ OpenAI API 호출
     │  └─ POST https://api.upstage.ai/v1/solar/chat/completions
     │  └─ 모델: solar-pro
     │  └─ 온도: 0.7
     │
     ├─ 응답 생성
     │  └─ "마음을 비우는 것이 첫 단계에요."
     │
     └─ 반환
        └─ {
             "answer": "...",
             "sources": ["source1.md", ...]
           }
  ↓
[Step 3] 메모리 저장
  └─ agent.memory.save_context(
       {"input": user_input},
       {"output": answer}
     )
  ↓
[Step 4] DB 저장 (FastAPI의 경우)
  └─ ChatMessage(
       session_id=sid,
       role="user",
       content=user_input
     )
     ChatMessage(
       session_id=sid,
       role="assistant",
       content=answer
     )
  ↓
[출력] answer를 사용자에게 반환
```

## 🎓 10. 기술 학습 포인트

### 10.1 핵심 개념

이 프로젝트에서 배울 수 있는 주요 기술:

#### 1. RAG (Retrieval-Augmented Generation)
```
개념:
- LLM의 약점: 학습 데이터 이후의 정보 모름
- 해결책: 외부 문서에서 관련 정보 검색
- 효과: 정확도 높은 답변 생성

구현:
검색 → 컨텍스트 조립 → LLM → 답변

이 프로젝트의 구현:
- 하이브리드 검색 (Vector + BM25)
- 컨텍스트 중심 프롬프팅
- 출처 추적
```

#### 2. 하이브리드 검색
```
문제: 단일 검색의 한계
- 벡터만: 의미적이지만 일부 누락
- 키워드만: 정확하지만 관련성 낮음

해결책: 두 방식 결합
1. 의미적 유사성 (Vector Embedding)
2. 키워드 일치 (BM25)
3. 결과 병합 (Ensemble)

이 프로젝트:
- ChromaDB (벡터)
- rank_bm25 (키워드)
- HybridRetriever (통합)
```

#### 3. 프롬프트 엔지니어링
```
패턴:
[System Prompt] 역할 정의
    ↓
[User Prompt] 작업 + 컨텍스트
    ↓
[LLM] 답변 생성

이 프로젝트:
- 시스템: "당신은 전우치이고..."
- 사용자: "[비급서]\n{context}\n\n[질문]"
- 마크다운 포매팅 강제
- 출처 언급 규칙
```

#### 4. 벡터 데이터베이스 & 임베딩
```
임베딩이란:
- 텍스트 → 숫자 벡터 (768차원)
- 의미 유사도 ≈ 벡터 거리

이 프로젝트:
- UpstageEmbeddings: 한국어 최적화
- ChromaDB: 지속적 저장소
- 의미 검색: 코사인 유사도
```
