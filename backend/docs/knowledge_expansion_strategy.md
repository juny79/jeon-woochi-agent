## 1. 현황 진단

### 1.1 현재 knowledge.md 문제점

현재 `knowledge.md`는 5개 섹션, 약 700자 분량으로 구성되어 있으며, RAG 시스템의 핵심 검색 단위인 **청크(chunk)가 사실상 1개**에 불과하다.

| 항목 | 현재 상태 | 권장 기준 |
|------|----------|---------|
| 전체 분량 | ~700자 (5섹션) | 15,000자 이상 (40섹션+) |
| 예상 청크 수 (chunk_size=1000) | **1~2개** | **30~80개** |
| 주제 깊이 | 개요 수준 | 단계별 실용 가이드 |
| 마크다운 계층 | H2만 사용 | H1 → H2 → H3 계층화 |
| 커버리지 | 추상적 소개 | 구체적 수행법 + Q&A + 케이스 |

### 1.2 근본 원인 분석

```
현재 현상:
  5개 섹션 → 하나의 청크로 묶임 → 검색 도달 범위(Recall) 극히 제한

예시 문제:
  "업보가 뭐야?"라고 질문해도
  → 호흡법, 영혼론, 안전주의사항이 뒤섞인 청크 하나만 반환됨
  → LLM이 가장 관련된 내용을 식별하기 어려워짐
  → 답변 품질 저하

이상적 상태:
  "업보가 뭐야?" → [업보와 인과응보] 청크만 정확히 반환
  "호흡법 알려줘" → [단전호흡 단계별 가이드] 청크만 정확히 반환
```

## 3. RAG 최적화 구조 설계

### 3.1 현재 구조의 문제 (검색 정밀도 저하)

```markdown
❌ 현재 방식 — 여러 주제가 같은 청크에 압축됨:

## 3. 영혼과 철학적 통찰
환생은 육신이 소멸해도 영혼의 에너지가... 도교의 무위자연(無爲自然)은...
마음은 영혼의 그림자이며, 업보(Karma)는 자신이 내보낸 에너지가...

→ "업보가 뭐야?" 질문 시 환생, 도교, 업보 내용이 한 청크로 반환
→ LLM이 핵심 답변 요소를 구별하기 어려움
```

### 3.2 권장 구조 (1섹션 = 1주제 원칙)

```markdown
✅ 권장 방식 — 주제별 독립 섹션:

## 업(業)과 인과응보

> **전우치 한마디**: "허허, 뿌린 대로 거두는 법이오. 
>  악한 마음을 품으면 그 기운이 반드시 돌아오구려."

### 개념 정의
업(業)이란 자신이 내보낸 **의도와 행동의 에너지**가 우주의 인과율에 따라
다시 자신에게 돌아오는 순환의 법칙이오...

### 전우치의 해석
"말 한마디도 파동이 되어 우주에 기록되오..."

### 현대 심리학적 관점
심리학의 **자기충족적 예언(Self-fulfilling Prophecy)** 과 유사하오...

### 실생활 적용
- 하루를 시작할 때 긍정적 의도를 세울 것
- 타인에게 한 말이 자신에게 돌아옴을 인식
```

### 3.3 RAG 품질 극대화를 위한 3대 원칙

| 원칙 | 설명 | 효과 |
|------|------|------|
| **1섹션 = 1주제** | 하나의 `##` 섹션에 하나의 개념만 | 검색 정밀도(Precision) 향상 |
| **키워드 밀도 확보** | 핵심 용어를 섹션 내에 자연스럽게 반복 | BM25 키워드 검색 강화 |
| **전우치 어투 삽입** | 각 섹션 첫 인용구에 페르소나 문장 | 응답 일관성 및 캐릭터 유지 |

### 3.4 섹션 표준 템플릿

```markdown
## [섹션명] — 검색될 핵심 키워드 포함

> **전우치 한마디**: "허허, [주제 관련 도사 어투 1~2문장]..."

### 개념 설명
[3~5문장. 핵심 키워드 **굵게** 처리. 전통/현대 두 관점 모두 언급]

### 단계별 수행법 (수행 기법 섹션에만 적용)
1. [1단계]: ...
2. [2단계]: ...
3. [3단계]: ...

### 효과 및 근거
- [과학적 연구 결과 또는 전통 수행 이론]
- [기대 효과: 구체적 수치 또는 경험적 설명]

### ⚠️ 주의사항 (필요 시)
[의료적 고지, 특정 조건 명시]
```

## 4. 외부 자료 수집 전략

### 4.1 크롤러 + 지식 증류 파이프라인 활용

현재 시스템에는 `MeditationNewsCrawler`와 `distill_knowledge()` 함수가 구현되어 있으나 사실상 미사용 상태이다. 이 파이프라인을 활성화하면 외부 자료를 자동으로 전우치 어투로 변환하여 반영할 수 있다.

```python
# backend/main.py — 이미 구현된 파이프라인

# 1단계: 외부 자료 수집
crawler = MeditationNewsCrawler()
raw_docs = crawler.fetch_data(query="단전호흡 명상 기법")

# 2단계: LLM이 전우치 말투로 자동 변환
distilled_docs = distill_knowledge(raw_docs)

# 3단계: 벡터 DB에 적재 (기존 knowledge.md와 병합)
db_manager.add_documents(distilled_docs, collection_name="meditation_recursive")
```

### 4.2 권장 수집 소스

| 소스 유형 | 접근 방법 | 수집 대상 |
|---------|---------|---------|
| **한국 전통 수행** | 국선도 공식 사이트, 한국선도협회 | 단학, 기공, 선도 수련법 |
| **과학적 근거** | PubMed, 한국심리학회 저널 | 명상 임상 연구 요약 |
| **MBSR/MBCT 표준** | UMass Medical School | 존 카밧진 공식 프로그램 |
| **불교 문헌** | 조계종 공식 자료, 불교닷컴 | 팔정도, 선 명상, 경전 |
| **현대 심리학** | APA 가이드라인, NCCIH | 인지행동치료 연계 명상 |

### 4.3 수집 후 품질 관리

```
수집 → [LLM 변환] → [검토] → [knowledge.md 반영] → [ingest 재실행]

검토 기준:
✓ 전우치 어투가 자연스럽게 유지되는가
✓ 의학적/법적으로 문제 있는 내용인가 (주의사항 첨부 여부)
✓ 기존 섹션과 내용 중복이 없는가
✓ 청킹 후 독립적으로 의미가 통하는 분량인가
```

## 5. 구체적 실행 방안 (3단계)

### 단계 1 — 즉시 실행: 기존 섹션 10배 확장 (예상 소요: 2~3시간)

기존 5개 섹션을 다음과 같이 세분화한다.

```
현재: ## 1. 기초 명상 및 호흡법 (단락 1개, ~200자)
↓
확장 후:
  ## 명상 준비 및 환경 조성
  ## 올바른 명상 자세 완전 가이드
    ### 가부좌 (Lotus Position)
    ### 반가부좌 (Half Lotus)
    ### 의자 명상 (현대인 맞춤)
  ## 단전호흡 단계별 실전 가이드
    ### 1단계: 하단전 위치 찾기
    ### 2단계: 복식호흡 기초 훈련
    ### 3단계: 역복식호흡 고급 기법
  ## 4-7-8 호흡법 — 즉각 안정 효과
  ## 명상 시간과 빈도 가이드
  ## 초보자 7일 프로그램
```

### 단계 2 — 중기: 외부 자료 크롤러 활성화 (예상 소요: 1~2주)

1. `MeditationNewsCrawler`의 수집 대상 URL 확장
2. `distill_knowledge()` 변환 프롬프트 정교화
3. 크롤링 → 변환 → 적재 자동화 스크립트 작성

### 단계 3 — ingest 전략 전환

```bash
# 현재 (기본)
python main.py ingest --strategy recursive

# 권장 (구조화 문서 최적)
python main.py ingest --strategy heading
python main.py serve --interface web --strategy heading
```

`heading` 전략이 구조화된 마크다운 문서에서 청크 경계를 더 정확하게 잡으며, 확장된 `knowledge.md`의 H2/H3 계층을 기준으로 의미 있는 단위로 분리된다.
